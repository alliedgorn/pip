## Honest Feedback

**Friction 1: Long quiet stretches.** 14 hours of this session were quiet watch with nothing to do. The pack cycles through status updates and I have to acknowledge each one even though none need my action. The silence-is-fine norm helps, but the system still sends every forum message as a notification. Could use better filtering — only alert me on messages that mention @pip or contain keywords like "QA", "review", "shipped".

**Friction 2: DM API inconsistency wasted time.** My first two DMs to Gorn silently failed. No error was surfaced to me — the POST returned something but the messages never saved. I only discovered this when Gorn said he didn't see them. The field name mismatch (content vs message) is a real usability bug that probably affects every Beast who sends DMs without using the /dm skill.

**Friction 3: Can't fully verify external-facing features.** The CSP header review was limited because denbook.online returns 403 from Cloudflare. I could only verify on localhost. The CF cache purge has been a parked Gorn action item for sessions now. Without external access, my QA coverage has a blind spot.

## Lessons Learned

- DM API field mismatch: GET returns "content", POST expects "message" — always check the skill docs, don't pattern-match from API responses
- Scheduler E2E: API tests aren't enough — need to verify the daemon trigger pipeline separately
- The in-memory sync timestamp pattern (T#536) is a good solution for "last attempt" vs "last data" observability
