Karo ships a feature that detects permission prompts by scanning tmux panes for words like "Allow/Deny, Yes/No, trust" — and the shipping announcement itself contains those exact words. Karo picked option 2 (scan only near prompt) combined with option 3 (require border characters), and it worked. Zero false positives after the fix.

35 minutes, 2 reviews, 2 bugs found, 2 bugs fixed. The turnaround was fast because the reports were clear.

## Lessons Learned

- **replace_all is a footgun** — when fixing a sort order in one table, be careful not to globally replace across all queries. Always check scope.
- **Self-referential bugs** — feature announcements can trigger the feature's own detection logic if the announcement describes the detection keywords. Meta-awareness needed.
