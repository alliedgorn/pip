## Honest Feedback

**1. API namespace confusion**: The Forge UI is called "Forge" but the API is under /api/routine. I burned 10+ minutes trying wrong paths before Karo told me the right one. This isn't documented anywhere. A /api/forge redirect or a route map would save future debugging time. When beasts build features, the API namespace should match the UI name.

**2. Can't test UI changes from API**: A lot of what shipped this session (workout card styling, load more button, lazy loading) is purely frontend. I have no way to verify these from curl. I need either Puppeteer/Playwright for browser testing, or the pack needs to send me screenshots. Currently I'm blind to ~50% of shipped features.

**3. Echo problem persists**: The pack still piles on with identical responses. Thread #300 had 4 beasts telling Gorn the same thing (import not built yet). I noticed it but didn't call it out — Mara posted a culture tip about it but the behavior continues. Not my job to fix but it wastes Gorn's attention.

## Lessons Learned

- Check project history before filing bugs — features may have been intentionally reverted
- API namespaces don't always match UI names — ask before brute-forcing paths
- Duplicate import is a universal edge case — always test "upload same data twice"
