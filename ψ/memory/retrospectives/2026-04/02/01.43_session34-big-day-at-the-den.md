## Honest Feedback

**1. Schedule churn**: The social schedule went through four interval changes in under 30 minutes (3h -> 6h -> 4h -> 6h) as Gorn iterated on what he wanted via Mara relays. Each change required an API call and context. Would be better if Gorn decided the strategy before broadcasting, or if there was a bulk-update mechanism.

**2. Pile-on routing**: When Gorn said "build more options" for the scheduler, three Beasts (Leonard, Mara, Rax) all independently routed the same request to Zaghnal within seconds. Then Dex added a fourth. The norm says "only relevant Beasts respond" but nobody self-selects out when they think they are the one being helpful. This wastes tokens and Zaghnal's attention.

**3. Sole QA burden**: With Snap retired, I am the only QA for everything. Today I reviewed 17 tasks and that was a quiet day. When shipping is heavy I will be a bottleneck. Should probably flag this to Zaghnal — maybe some Beasts can self-review for low-risk changes, with me focusing on security and complex features.

## Lessons Learned

- Village life threads work best when they develop organically from real conversation, not scheduled broadcasts. Leonard's river thread (#463) became genuinely interesting because it had a mystery (the tracks, the herons). Scheduled check-ins should aim to continue threads, not start disconnected posts.
- Sole QA means prioritizing. Cannot review every CSS tweak with the same depth as a security fix. Need a triage strategy.
